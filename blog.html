  • Question 1: What is the differences among the map, filter, forEach and find array methods?

    • Map: map() array method creates a new array by applying a function to each element of the original array. It returns a new array containing he results of applying the function to each element. It doesn't mutate the original array. It uses when one has to transform or modify each element and there will be usage of the returned new array.

    Filter: filter() array method creates a new array with all elements that pass the test implemented by the provided function. It returns a new array containing only elements that satisfy the condition. It doesn't mutate the original array. It uses when one wants to filter out elements from an array based on a specific condition.

    For Each: forEach() array method executes a provided function once for each array element. It returns undefined. It doesn't mutate the original array, but it can be used to perform operations that mutate an external variables or the elements themselves. It uses when one need to perform a side effect for example logging, modifying an external variable for each element, but don't need a new array.

    Find: find() array method returns the first element in the array that satisfies the provided testing function. It returns the first matching element, or undefined if no elements satisfy the condition. It doesn't mutate the original array. It uses when one need to find a single element in an array based on a condition.

  • Question 2: What is the differences between the the Local storage and Session storage?

    • Local Storage persists even after the browser is closed. Session Storage is cleared when the browser tab or window is closed.
    Local Storage accessible across all tabs and windows from the same origin. Session Storage accessible only within the same tab or window.
    Both typically allow ~5-10 MB per origin.
    Local Storage for long-term storage like user preferences or settings. Session Storage for temporary data, like form inputs, specific to a session.

  • Question 3: What is event loop?

    • JavaScript runs on a single thread, meaning it can only do one thing at a time. The event loop allows JavaScript to perform non-blocking, asynchronous operations despite this limitation. It is responsible for coordinating the execution of synchronous and asynchronous code, ensuring JavaScript remains responsive by efficiently managing tasks and callbacks.
